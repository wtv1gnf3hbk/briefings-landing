How they work

Each briefing is its own GitHub repository. A scheduled GitHub Action scrapes a curated list of regional sources, sends the raw items to Claude with bureau-specific style rules, runs the draft through a validator script that catches banned phrases, missing attribution, and structural drift, then publishes the result to GitHub Pages. A refresh button on each page triggers a fresh build on demand.

A management layer monitors all 13 repos every two hours, applies known-pattern fixes when trusted recipes match, and escalates novel failures.
